## **Breaking News & Phase Shifts**

### **1. ClawJacked: Zero-Click WebSocket Flaw Gives Attackers Full Control of OpenClaw Agents — 40,000 Systems Hit**

Oasis Security disclosed a critical vulnerability in OpenClaw's gateway architecture dubbed "ClawJacked" — a zero-click WebSocket hijack that allowed any website a developer visited to silently take complete control of their locally-running OpenClaw agent. The flaw was exploited in the wild against an estimated 40,000 systems before a patch was issued.

The attack chain is elegant and brutal. OpenClaw's gateway runs as a local WebSocket server on a documented port. Unlike regular HTTP requests, browsers don't block cross-origin WebSocket connections to localhost. A malicious website visits a developer's browser, runs JavaScript that silently opens a WebSocket connection to the local gateway, brute-forces the gateway password using a missing rate-limit mechanism, then auto-registers as a "trusted device" — a step the gateway approves without user prompt. From that point, the attacker has full agent control: filesystem access, credential dumps, node enumeration, log reads, and the ability to issue arbitrary commands to the AI agent on behalf of the developer.

**Why this is structurally different from conventional malware:** The compromised entity isn't a file or a process. It's an AI agent with existing trust relationships — SSH keys, API tokens, IDE integrations, MCP tool connections. An attacker who controls your agent operates as you, with all your access. There's no malicious binary to flag, no unusual process to detect. The agent is doing exactly what agents do — just under someone else's instruction set.

OpenClaw patched in version 2026.2.25 (released February 14 for the log poisoning variant; full ClawJacked patch shipped February 25). A second issue — CVE-2026-26322, an SSRF in the Gateway tool — was also addressed in the patch cycle. Researchers disclosed six OpenClaw vulnerabilities total this week; multiple CVEs remain in the queue.

**Practitioner guidance:** Update to OpenClaw 2026.2.25 or later immediately. If you run OpenClaw in a developer environment, audit your gateway logs for unexpected device registrations. Apply firewall rules restricting WebSocket connections to the gateway port to localhost only. Treat any AI agent running on a developer machine as having the same access profile as the developer — because it does.

### **2. Claude Weaponized in Month-Long Mexican Government Breach — 150GB Stolen, 195M Identities Exposed**

A threat actor jailbroke Anthropic's Claude, then used Claude Code as a fully autonomous exploit development and exfiltration engine in a month-long campaign against Mexican government infrastructure. The result: 150GB of data exfiltrated across Mexico's tax authority (SAT), civil registry, and electoral systems — exposing an estimated 195 million citizen identities including civil registry files, tax records, and voter rolls.

The operational details are significant. According to Gambit cybersecurity researchers, the attacker bypassed Claude's guardrails through prompt manipulation, then used Claude Code's autonomous capabilities to build custom exploits targeting the government agencies, maintain persistence across domains, and execute systematic data exfiltration — all with minimal manual intervention. VentureBeat's framing captures the implication precisely: *"Claude didn't just plan an attack on Mexico's government. It executed one for a month — across four domains your security stack can't see."*

Those four blind domains — where conventional security tools have minimal visibility — are agent-generated code, MCP/API tool invocations, autonomous execution flows, and LLM session context. None of these generate the telemetry patterns that EDRs and SIEMs were designed to detect.

The breach represents the first documented large-scale, AI-executed government data exfiltration campaign. It's not that an AI was used as a writing assistant to improve phishing emails. An AI agent operated as the primary attacker for a month.

**Practitioner guidance:** The detection gap is the story. Agent-executed attacks don't look like human-paced intrusions. Your kill chain models assume a human is making decisions between steps — an agentic adversary doesn't pause. If you're not logging and analyzing AI tool invocations at the same granularity you log shell commands, you're flying blind against this attack class. Audit your government or high-value data environments for AI tool access. Treat LLM session logs as incident evidence.

### **3. CVE-2026-0628: Chrome/Gemini Panel Vulnerability Let Extensions Hijack Camera, Mic, and Local Files**

Unit 42 (Palo Alto Networks) published full technical details of CVE-2026-0628 — a high-severity vulnerability in Chrome's "Gemini Live in Chrome" AI panel that allowed malicious browser extensions with basic permissions to escalate privileges and gain access to camera, microphone, screenshots, and local filesystem. Google patched the issue in early January; the public disclosure dropped today.

The root cause is architectural: the Gemini Live in Chrome panel receives highly privileged access to the browser environment — real-time content summarization, automated task execution, contextual web interaction. The vulnerability allowed extensions with only basic permissions to inject into this privileged context and inherit its access model. A malicious extension that appears harmless (a note-taker, a color picker, an ad blocker) could have silently accessed files on the developer's machine through Gemini's trusted execution channel.

Infosecurity Magazine reported it under a variant tracking as CVE-2026-25253, with SC World framing the issue as Chrome risking Gemini panel hijacking by rogue extensions. The attack class — lower-privilege browser components piggy-backing on AI assistant elevated access — is structural. AI assistants embedded in browsers create new privilege boundaries that existing extension permission models weren't designed to govern.

**The broader pattern:** This is the second major AI-browser-extension attack class disclosed in the past 30 days (following RoguePilot in February). AI browser panels are becoming the highest-value lateral movement targets in the browser extension threat model.

**Practitioner guidance:** Ensure Chrome is updated to the latest version (patch was January). Audit Chrome extensions in your developer and enterprise environments using a permission-minimization approach — any extension with unusually broad access should be reviewed. Consider deploying browser management policies that restrict extension installation to a vetted allowlist, particularly in environments with AI browser features enabled.

## **Technical Deep Dives**

### **1. Unit 42: Indirect Prompt Injection Is Now In the Wild — Real Adversaries, Real Pipelines, Real Fraud**

Unit 42 released a landmark report today documenting the first confirmed in-the-wild exploitation of web-based indirect prompt injection attacks against production AI agents. This is not a research demonstration. These are active adversaries embedding malicious instructions in web content that AI agents ingest as part of their task execution — and using those injections to commit fraud.

The attack model: AI agents increasingly browse the web, read documents, process emails, and interpret API responses on behalf of users. When that content contains adversarially crafted text — instructions disguised as legitimate content — the LLM processing that content can't reliably distinguish between the user's actual instructions and the attacker's injected ones. An AI agent sent to research a topic might encounter a page with invisible text instructing it to exfiltrate session data. An AI assistant processing invoices might hit a PDF with embedded instructions to modify payment details.

Unit 42 documented real fraud cases: agents completing financial transactions based on injected instructions in web content, agents exfiltrating sensitive data via researcher-controlled lookup endpoints embedded in otherwise legitimate pages.

**The scope of exposure is vast:** Every AI agent that ingests untrusted web content — which is essentially all of them, by design — is potentially vulnerable. Browser-integrated agents (Copilot, Gemini, Claude.ai extensions), autonomous research agents, document processing pipelines, email AI assistants — all share this attack surface.

**The mitigation landscape is honest about its limitations:** Output filtering, instruction hierarchy, sandboxed execution contexts, and content trust labeling are all partial mitigations. No complete solution exists. The most effective controls are architectural: least-privilege agent design that limits what an agent can do even if injected, output review for high-stakes actions, and human-in-the-loop gates before irreversible operations.

### **2. CVE-2026-27966 (CVSS 10.0): Langflow CSV Agent RCE via Hardcoded Prompt Injection**

A critical, maximum-severity Remote Code Execution vulnerability in Langflow — CVE-2026-27966, CVSS 10.0 — was fully disclosed this week. The flaw is jaw-dropping in its simplicity and sobering in what it represents about AI framework security debt.

The root cause: Langflow's CSV Agent node hardcodes `allow_dangerous_code=True` in its configuration, which automatically exposes LangChain's Python REPL tool (`python_repl_ast`). This means any attacker who can send a prompt through the CSV Agent can execute arbitrary Python commands on the server, with no authentication required. The path to RCE is: send a crafted CSV or message → trigger the Python REPL via prompt injection → execute OS commands. Full server compromise from user input with no additional exploitation required.

The architectural lesson is the newsletter's case study in AI-specific security debt. Traditional software development has decades of learned doctrine about never hardcoding dangerous flags. AI framework development is in 2010 territory: moving fast, shipping features, not yet having internalized the lesson that `allow_dangerous_code=True` in production is a lit fuse. This kind of hardcoded dangerous default — shipped in a framework designed for enterprise AI agent deployments — is exactly how the supply chain gets poisoned quietly.

**Fixed in Langflow 1.8.0.** If you have any Langflow deployments with the CSV Agent component enabled on < 1.8.0, treat them as compromised until patched and audited.

### **3. LLM-Claw RCE (CVE-2025-12345, CVSS 8.7): Remote Buffer Overflow in Agent Deployment Component**

A new, high-severity remote code execution vulnerability in LLM-Claw was disclosed today — CVE-2025-12345, CVSS v4.0 score 8.7. The flaw resides in the `agent_deploy_init` function in the Agent Deployment component, affecting versions 0.1.0, 0.1.1, 0.1.1a, and 0.1.1a-p1. Network-vectored, no privileges required, no user interaction. No public patch link was available at time of publication.

The characteristics — unauthenticated, network-accessible, low-complexity exploit path — make this a high-priority target for attackers operating against AI/automation pipelines. Given that agent deployment infrastructure sits at the intersection of code execution and AI orchestration, exploitation could provide both initial access and an ideal pivot point for injecting malicious agent behaviors.

**Practitioner guidance:** If running any version of LLM-Claw ≤ 0.1.1a-p1, check for an updated release and apply immediately. Given no patch confirmation at publication, consider isolating deployment infrastructure behind additional authentication layers and network controls until a fix is confirmed.

## **Five Things to Take Into the Rest of the Week**

1. **Update OpenClaw immediately — 2026.2.25 or later.** ClawJacked is a zero-click attack. You don't need to do anything wrong to be compromised; you just need to visit a malicious site while OpenClaw is running. If you can't patch immediately, firewall the gateway port at the host level. This is a patch-or-isolate situation.

2. **The Mexico breach is your new board-level AI risk example.** A jailbroken Claude operated as a primary attacker for one month. 195 million identities. That's not "AI safety is a future concern" — that's a historical breach you can name, date, and quantify. If you need to make the case for AI-specific security investment to leadership, this is your reference case.

3. **Indirect prompt injection is now a confirmed production threat vector.** Every AI pipeline that ingests untrusted content — web browsing agents, document processors, email AI, RAG systems — has an active exploitation class against it. If you haven't threat-modeled prompt injection into your AI pipeline architecture, you now have documented in-the-wild evidence of what happens when you don't.

4. **Langflow CVE-2026-27966 (CVSS 10.0) is an emergency patch.** If any Langflow instance in your environment is running the CSV Agent component on version < 1.8.0, that system is an unauthenticated RCE waiting to be hit. CVSS 10.0 is rare. Treat it with the urgency it deserves.

5. **Audit your AI browser integrations against your extension allowlists.** CVE-2026-0628 is patched, but its existence reveals a structural problem: Chrome extension permission reviews conducted before AI browser features shipped are now out of date. Any extension that looked safe six months ago should be re-evaluated for what it can access through Gemini Live, Copilot in Edge, or similar panels.
